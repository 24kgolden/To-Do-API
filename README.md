## вңЁ CaracterГӯsticas

| Endpoint | DescripciГіn |
|----------|-------------|
| `GET /api/tasks` | Lista todas las tareas con sus datos completos |
| `?completed=true/false` | Filtra tareas por estado de completado |
| `?priority=HIGH/MEDIUM/LOW` | Filtra tareas por nivel de prioridad |
| `?search=texto` | BГәsqueda de tareas por tГӯtulo (insensible a mayГәsculas) |
| `POST /api/tasks` | Crea una nueva tarea con validaciones automГЎticas |
| `PATCH /{id}/toggle` | Alterna el estado completado de una tarea |
| `DELETE /{id}` | Elimina una tarea, devuelve `404` si no existe |

## рҹ“Ұ Estructura del modelo `Task`

Cada tarea tiene los siguientes campos:

| Campo | Tipo | DescripciГіn |
|-------|------|-------------|
| `id` | `Long` | Identificador Гәnico (autogenerado) |
| `title` | `String` | TГӯtulo obligatorio, mГЎx. 150 caracteres |
| `description` | `String` | DescripciГіn opcional, mГЎx. 500 caracteres |
| `completed` | `boolean` | Estado de completado (por defecto `false`) |
| `priority` | `enum` | Prioridad: `LOW`, `MEDIUM`, `HIGH` |
| `createdAt` | `LocalDateTime` | Fecha de creaciГіn (automГЎtica) |
| `updatedAt` | `LocalDateTime` | Fecha de Гәltima modificaciГіn (automГЎtica) |

> вҡ пёҸ El campo `title` es obligatorio. Si se omite o se envГӯa vacГӯo, la API responde con `400 Bad Request` y un mensaje descriptivo del error.
